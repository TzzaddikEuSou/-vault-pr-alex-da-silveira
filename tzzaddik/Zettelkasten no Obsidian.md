[[🧩 MODELO 1 — Zettelkasten no Obsidian]]

Você quer um vault que respire organização. Um cérebro externo que não seja um depósito de PDFs órfãos, mas uma cidade viva com ruas, pontes e placas 🧠🏙️

Aqui vão **modelos prontos para colar no seu Obsidian**, tanto para **Zettelkasten** quanto para **PARA**, já estruturados para uso imediato.

---

# 🧩 MODELO 1 — Zettelkasten no Obsidian

Inspirado no método de Niklas Luhmann.

## 📂 Estrutura de Pastas

```
00 - Inbox
01 - Literatura
02 - Notas Permanentes
03 - Índices (MOCs)
04 - Projetos
```

---

## 📝 Template 1 — Nota de Literatura

Use para resumir livros, artigos, sermões, cursos.

```
---
tipo: literatura
autor:
fonte:
ano:
tags: []
---

# Título da Obra

## 📌 Ideia Central
Resumo da tese principal em 3–5 linhas.

## 🧠 Insights Importantes
- Insight 1
- Insight 2
- Insight 3

## 💬 Citações Relevantes
> "Trecho importante"

## 🔗 Conexões
- [[Nota Permanente relacionada]]
```

---

## 🧠 Template 2 — Nota Permanente (o coração do sistema)

Cada nota deve conter **uma única ideia clara**.

```
---
tipo: permanente
criado: {{date}}
tags: []
---

# Título claro e específico

## 🧩 Ideia

Explique o conceito com suas próprias palavras.
Se não consegue explicar, ainda não entendeu.

## 🔗 Conexões
- Relaciona-se com [[Outra Nota]]
- Contrasta com [[Nota Y]]

## 📚 Origem
Baseado em [[Nota de Literatura X]]
```

---

## 🗺️ Template 3 — MOC (Map of Content)

Funciona como um mapa rodoviário das ideias.

```
---
tipo: indice
---

# MOC — Tema Central

## Conceitos Fundamentais
- [[Nota 1]]
- [[Nota 2]]

## Aplicações Práticas
- [[Nota 3]]

## Perguntas em Aberto
- [[Nota 4]]
```

📌 Dica: MOCs são faróis no Graph View. Eles iluminam o caos.

# 📦 MODELO 2 — PARA no Obsidian

Método criado por Tiago Forte.

PARA significa:

- **P**rojects (Projetos ativos)
    
- **A**reas (Áreas de responsabilidade contínua)
    
- **R**esources (Recursos)
    
- **A**rchives (Arquivados)
    

---

## 📂 Estrutura de Pastas

```
01 - Projects
02 - Areas
03 - Resources
04 - Archives
```

---

## 🚀 Template — Projeto

```
---
tipo: projeto
status: ativo
prazo:
area:
tags: []
---

# Nome do Projeto

## 🎯 Objetivo
Definir claramente o resultado final desejado.

## 📌 Próximas Ações
- [ ] Tarefa 1
- [ ] Tarefa 2

## 📚 Recursos Relacionados
- [[Nota Recurso X]]

## 📝 Notas do Projeto
Anotações livres aqui.
```

---

## 🏛 Template — Área

Áreas nunca terminam. São responsabilidades contínuas.

```
---
tipo: area
---

# Nome da Área

## 📖 Propósito
Por que essa área existe?

## 📊 Indicadores
Como sei que estou indo bem?

## 🔗 Projetos Ativos
- [[Projeto 1]]
- [[Projeto 2]]
```

---

## 📚 Template — Recurso

```
---
tipo: recurso
tema:
tags: []
---

# Tema ou Assunto

## 📌 Resumo
Descrição clara do conteúdo.

## 🔗 Conexões
- [[Projeto relacionado]]
- [[Área relacionada]]
```

---

# 🧠 Zettelkasten vs PARA — Quando usar cada um?

|Zettelkasten|PARA|
|---|---|
|Construção de conhecimento|Gestão prática da vida|
|Ideias atemporais|Execução de tarefas|
|Rede conceitual|Organização operacional|

💡 Muitos usuários combinam os dois:

- PARA para estrutura macro.
    
- Zettelkasten dentro de Resources.
